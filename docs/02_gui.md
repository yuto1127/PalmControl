# GUIの使い方

GUIはタブ切り替え式です。研究用途として、まず「見える化（プレビュー）」と「OS操作」を分離できるようにしています。

## 設定タブ

- **YAML再読込**: 手編集した `settings.yaml` をディスクから読み直します
- **OS操作を有効化（危険）**: ONのときだけ、実際にマウス操作が発行されます
- **アプリケーション終了**: アプリを終了します

設定フォームは項目数が多いため、**縦スクロール**で全項目にアクセスできます。

## モーションテストタブ

- **プレビュー表示（重い）**: 映像とオーバーレイ表示を行います（CPU負荷が増えます）
- **OS操作を有効化（危険）**: 実機操作のON/OFF

ステータス表示は、検出（vision）と制御（control）の情報を分けて出します。`mode=None` でも制御状態が進むことがあるのは、検出モードと接触（pinch）系の判定が別軸だからです。

## ログタブ

ログファイルの先頭付近（最新側）を一定行数表示します。

## マニュアルタブ

`docs/*.md` を読み込み、ファイル名順に連結表示します。
