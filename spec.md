PalmControl プロジェクト仕様書

1. プロジェクト概要
コンセプト: 「手のひらでPCを支配する」
カメラ映像からハンドトラッキングを行い、マウス操作やジェスチャーによるショートカット実行を可能にする。研究用途として、各機能の単体検証が可能で、かつ実用的なGUIを備えたクロスプラットフォームツールを目指す。

2. 技術スタック
OS: macOS (M1/M2/M3 Apple Silicon対応), Windows 10/11

言語: Python 3.10+

ライブラリ:

解析: MediaPipe (Hand Landmarking)

画像処理: OpenCV

OS操作: PyAutoGUI, pynput

GUI: PyQt6 (または PySide6)

設定管理: PyYAML

3. システムアーキテクチャ
処理負荷を抑え、リアルタイム性を確保するために「マルチスレッド構成」を採用する。

Main Thread (GUI): 設定、ログ表示、常駐トレイ管理。

Worker Thread (Vision Engine): カメラ取得、MediaPipe解析、座標変換。

Controller: OSへの入力信号送信。

4. 主要機能
4.1 ハンドトラッキング・マウス機能
座標マッピング: 指定したカメラ範囲（ROI）を画面解像度に変換。

スムージング: 指数移動平均（EMA）フィルタによるポインタの震え抑制。

クリック判定: 指先同士の距離（ユーカリ距離）に基づく判定。

緊急停止: 特定のジェスチャーまたはホットキーによる強制終了。

4.2 GUIアプリケーション
設定パネル: 解像度、FPS、感度、しきい値のリアルタイム変更。

モーションテスト: 骨格描画（Landmarks）をオーバーレイしたデバッグ用プレビュー。

動作ログ: 認識成功率や処理時間（ms）の記録・表示。

バックグラウンド動作: 最小化時にシステムトレイ（メニューバー）へ格納。

4.3 ジェスチャー定義の詳細（実装ロジック）
各ジェスチャーは、MediaPipeのランドマークID（親指:4, 人差し指:8, 中指:12）の距離および
指の屈曲状態に基づいて判定する。

4.3.1. カーソル操作 (Pointer Move)指の状態: 人差し指（8）と中指（12）が伸展（伸びている）、かつ薬指と小指が屈曲（曲がっている）状態。
ポインター座標: 人差し指（8）と中指（12）の中間点をカーソル座標として採用。
メリット: 指1本よりも検出面積が広いため、遠距離や低解像度でもトラッキングが安定する。

4.3.2. クリック・ドラッグ系（親指のコンタクト判定）※「人差し指と中指を立てた状態」を維持したまま、親指（4）を人差し指（8）の付け根または中間に接触させる動作で判定。
ドラッグ＆ドロップ (Drag):判定: 親指と人差し指（または中指）の距離が一定値（click_threshold）以下の間、マウスボタンを Down 状態に保持。
解除: 指を離すと Up 状態に。
左クリック (Left Click):
判定: 上記の「コンタクト（接触）」が 2回連続（ダブルタップ） で行われた場合に発火。
右クリック (Right Click):判定: 上記の「コンタクト（接触）」が 3回連続（トリプルタップ） で行われた場合に発火。

4.3.3. スクロール (Scroll)指の状態: 全ての指（4, 8, 12, 16, 20）が伸展している「パー」の状態。
判定: 手のひらの中心（Landmark 0, 5, 17の平均など）のY座標の変化量を計測。
動作: 前回のフレームからの垂直移動量をスクロール量に変換して送出。

4.3.4. 認識精度と安定性のための制御アルゴリズム　
実装においては、誤動作防止と操作感向上のため以下の3つのロジックを必須とする。
① 状態の排他制御 (Finger Count Priority)認識エンジンは、フレームごとにまず「伸展している指の総数」を確認し、トップレベルでモードを分岐させる。
5本（パー）: スクロールモードとして判定。この時、マウス座標の移動処理は無視する。
2本（人差し指・中指）: マウス/クリックモードとして判定。これにより、マウス移動中に意図せずスクロールが暴発するなどの干渉を防ぐ。

② タップ判定のバッファ管理 (Action Queueing)連続タップ（左/右クリック）を正確に識別するため、controller.py 内に動作履歴を保持するキューを実装する。親指の接触（コンタクト）と離脱イベントをタイムスタンプと共に記録。settings.yaml の tap_interval_ms（例: 300ms）以内にイベントが完結したかを確認し、クリック回数を確定させる。単発の長押し（ドラッグ開始）と、素早いタップ（クリック）を時間軸で明確に判別する。

③ クリック時の座標固定 (Cursor Anchoring)親指を接触させる動作に伴う微細な手ブレ（意図しないカーソル移動）を抑制する。親指が人差し指に接近している（タップの予兆）と判定された瞬間、および接触中の N（任意の数）フレーム間は、マウス座標の更新を一時停止、または平滑化フィルタの重みを極端に強化する。これにより、微細なUIパーツ（ボタンやリンク）のクリック精度を向上させる。

5. 開発・実装ルール (研究用ガイドライン)
研究としての透明性を高めるため、以下のルールを厳守する。

5.1 設定の一括管理 (config/settings.yaml)
すべての定数は外部ファイルで管理し、コードを書き換えずに調整可能とする。

カメラパラメータ（Device ID, Width, Height, FPS）

解析パラメータ（Confidence Score, Frame Skip）

制御パラメータ（Sensitivity, Smoothing Factor, Thresholds）

5.2 モジュール化とテストコード
機能ごとにファイルを分割し、それぞれに対して tests/ ディレクトリ内に単体テストコードを作成する。

test_camera.py: 映像取得の安定性確認。

test_detection.py: 手の骨格検知精度の検証。

test_control.py: マウス移動・クリックの挙動確認。

5.3 ドキュメンテーション
コメントアウト: 全ての関数に日本語のdocstringを付与し、複雑なロジックには「なぜその処理が必要か」の意図をコメントで残す。

ログ管理: ログはJSON形式で記録し、最新データが上位に来るよう管理する。また、ディスク容量を圧迫しないよう、最大ファイルサイズ（例：10MB）を超えた場合に自動でバックアップを作成し、古いデータを整理するローテーション機能を備える。

7. 制約事項
負荷抑制:

最小化（バックグラウンド動作）時はプレビュー描画を完全に停止し、CPU/GPU負荷を最小限に抑える。

解析解像度を意図的に下げ（640x480推奨）、処理負荷を最適化する。

OS権限への対応:

macOSにおける「アクセシビリティ権限」の取得が必要な旨を設計・マニュアルに含める。

安全停止 (Fail-safe):

プログラムが暴走した場合に備え、特定の物理キー（例：Esc長押し）または特定の緊急停止ジェスチャーで即座にプロセスを終了できる機能を必須とする。

クロスプラットフォーム:

パス指定やライブラリ挙動において、WindowsとMacの差異を吸収する抽象化を行う。

スケーラビリティ: 将来的に新しいジェスチャーを容易に追加できる拡張性を持たせる。

設定変更の容易性:システム動作に必要な設定情報やパラメータは1つのファイルで管理し、各プログラムファイルはそこを参照し値を読み込んで使用する。

座標変換の最適化: カメラ映像の全域を使用せず、中心部の特定領域（操作有効エリア）を設けることで、少ない手の動きで画面全域をカバーできるマッピングロジックを実装する。

パラメータの即時反映: GUIで変更された設定値は、アプリケーションを再起動することなくリアルタイムで動作に反映させる。
